# FR-0039: Synthesis stub propagation

## Observed behavior

`/opt/workspace/runtime/.meta/LATEST_SYNTHESIS` has pointed to
`cross-cutting-2026-04-28T03-25-05Z.md` (67 bytes — just the path to itself) for 4+
consecutive synthesis cycles. The synthesis translator is generating carry-forward
proposals based on this stub, inflating the INBOX without any real synthesis content.

## Evidence

```
wc -c /opt/workspace/runtime/.meta/cross-cutting-2026-04-28T03-25-05Z.md
# → 67 /opt/workspace/runtime/.meta/cross-cutting-2026-04-28T03-25-05Z.md

cat /opt/workspace/runtime/.meta/cross-cutting-2026-04-28T03-25-05Z.md
# → /opt/workspace/runtime/.meta/cross-cutting-2026-04-28T03-25-05Z.md
```

The file contains only its own path. This is the stub output pattern: the Claude invocation
returned nothing useful and synthesize.sh wrote whatever came back, then unconditionally
updated LATEST_SYNTHESIS.

## Root cause

`synthesize.sh` (line ~89) unconditionally runs `ln -sf "$OUTPUT_FILE" "$LATEST_SYNTHESIS"`
regardless of output size. A failed or empty synthesis invocation produces a stub file
that gets promoted to LATEST_SYNTHESIS. The synthesis translator then reads the stub,
finds nothing, and generates carry-forwards from the prior cycle, inflating INBOX.

## Fix (proposed in INBOX 5x, blocked by Tier-C restriction)

Add a size gate in `synthesize.sh` before the LATEST_SYNTHESIS pointer update:

```bash
OUTPUT_SIZE=$(wc -c < "$OUTPUT_FILE")
if [ "$OUTPUT_SIZE" -lt 500 ]; then
  echo "synthesize: WARNING — output ${OUTPUT_SIZE} bytes, not updating LATEST_SYNTHESIS" >&2
  exit 0
fi
ln -sf "$OUTPUT_FILE" "$WORKSPACE_LATEST_SYNTHESIS_PTR"
```

This prevents corrupt pointers from propagating. The root cause (why synthesis sometimes
produces empty output) is a separate investigation.

## Impact

- INBOX carries 15+ stale proposals generated by the translator reading stubs
- Attended session cannot detect "no new synthesis" from "bad synthesis" — both look the same
- Meta-loop effectively broken for 4+ cycles

## Status

Open. Fix requires attended session with write access to `scripts/lib/synthesize.sh`.
