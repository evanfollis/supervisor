# Fix reflect.sh argument ordering bug

**Root cause (verified 15:24Z):** `reflect.sh` lines 193–196 place the `--` pathspec separator before the `-m` flag. Git's `--` terminates option parsing, so everything after it is treated as a pathspec. The `-m` flags and message strings are parsed as filenames, producing `error: pathspec '-m' did not match any file(s) known to git`.

**Direct verification:**
1. `/var/log/workspace-reflect.log` contains `reflect[context-repository]: WARNING — CURRENT_STATE.md commit failed` for every project, every cycle going back weeks.
2. `git -C /opt/workspace/projects/context-repository commit --only -- CURRENT_STATE.md -m "test"` → fails with `error: pathspec '-m'` (exit 1).
3. `git -C /opt/workspace/projects/context-repository commit --only --dry-run -m "test" -- CURRENT_STATE.md` → succeeds (exit 0).

**Fix (reorder arguments only):**

In `/opt/workspace/supervisor/scripts/lib/reflect.sh`, replace lines 193–196:

```bash
# CURRENT CODE (broken):
git -C "$PROJECT_DIR" commit --only -- CURRENT_STATE.md \
     -m "reflect: auto-update CURRENT_STATE.md ${ISO_NOW}" \
     -m "Generated by scripts/lib/reflect.sh ..." \
     -m "session_id=reflect-${PROJECT}-${ISO_NOW}"
```

With:

```bash
# FIXED (move -- CURRENT_STATE.md to end):
if git -C "$PROJECT_DIR" commit --only \
     -m "reflect: auto-update CURRENT_STATE.md ${ISO_NOW}" \
     -m "Generated by scripts/lib/reflect.sh after the ${PROJECT} reflection session." \
     -m "session_id=reflect-${PROJECT}-${ISO_NOW}" \
     -- CURRENT_STATE.md \
     >/dev/null 2>&1; then
```

**Impact:** Unblocks CURRENT_STATE.md commits across atlas, command, context-repository (20 uncommitted passes in context-repo alone). Closes a 29-day drift where the auto-commit code existed but never fired.

The evidence (WARNING messages) has been accumulating in plaintext for 29 days.
