# Fix reflect.sh argument ordering

**Type:** Shared primitive fix.

**What:** At `scripts/lib/reflect.sh:193`, change:
```bash
if git -C "$PROJECT_DIR" commit --only -- CURRENT_STATE.md \
     -m "reflect: auto-update CURRENT_STATE.md ${ISO_NOW}" \
```

to:
```bash
if git -C "$PROJECT_DIR" commit --only \
     -m "reflect: auto-update CURRENT_STATE.md ${ISO_NOW}" \
     -m "Generated by scripts/lib/reflect.sh after the ${PROJECT} reflection session." \
     -m "session_id=reflect-${PROJECT}-${ISO_NOW}" \
     -- CURRENT_STATE.md \
```

**Why:** Unblocks CURRENT_STATE.md auto-commit across 5 projects. The `--` path separator must follow all option flags per git's argument parsing. Currently the `-m` flags after `--` are treated as file paths.

**Impact:** All reflected projects (atlas, skillfoundry-harness, skillfoundry-valuation, skillfoundry-researcher, command). Argument reorder only — no behavioral change to the commit itself.
