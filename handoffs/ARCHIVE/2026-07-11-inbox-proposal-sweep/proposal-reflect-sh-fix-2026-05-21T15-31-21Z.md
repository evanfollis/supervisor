# Fix reflect.sh argument ordering bug

**Type:** Shared primitive fix.

**Blast radius:** All reflected projects (automatic CURRENT_STATE.md auto-commits will resume across context-repository, command, atlas, supervisor).

**Change location:** `/opt/workspace/supervisor/scripts/lib/reflect.sh:193`

**Current state (VERIFIED):**
```
    if git -C "$PROJECT_DIR" commit --only -- CURRENT_STATE.md \
         -m "reflect: auto-update CURRENT_STATE.md ${ISO_NOW}" \
         -m "Generated by scripts/lib/reflect.sh after the ${PROJECT} reflection session." \
         -m "session_id=reflect-${PROJECT}-${ISO_NOW}" \
```

**Bug:** `-- CURRENT_STATE.md` appears before the `-m` flags, which causes git to treat `-m` as a pathspec rather than a flag. The commit silently fails, and CURRENT_STATE.md changes accumulate across reflection passes without committing.

**Fix:** Move `-- CURRENT_STATE.md` to after the three `-m` flags:
```
    if git -C "$PROJECT_DIR" commit --only \
         -m "reflect: auto-update CURRENT_STATE.md ${ISO_NOW}" \
         -m "Generated by scripts/lib/reflect.sh after the ${PROJECT} reflection session." \
         -m "session_id=reflect-${PROJECT}-${ISO_NOW}" \
         -- CURRENT_STATE.md \
```

**Evidence of impact:**

- **command**: 9th cycle uncommitted
