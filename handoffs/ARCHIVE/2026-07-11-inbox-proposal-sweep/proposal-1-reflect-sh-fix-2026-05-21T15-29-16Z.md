# Fix reflect.sh argument ordering bug

**Type:** Shared primitive fix.

**Change:** `scripts/lib/reflect.sh:193` — move `-- CURRENT_STATE.md` to after the three `-m` flags.

**Blast radius:** All reflected projects (automatic).

**Status:** Exceeds 3-cycle gate. INBOX saturation suppression active. Suppression recorded.

## Context

The bug in reflect.sh prevents CURRENT_STATE.md from being committed automatically across reflected projects. The current form at line 193 is:

```bash
if git -C "$PROJECT_DIR" commit --only -- CURRENT_STATE.md \
     -m "reflect: auto-update CURRENT_STATE.md ${ISO_NOW}" \
     -m "Generated by scripts/lib/reflect.sh after the ${PROJECT} reflection session." \
     -m "session_id=reflect-${PROJECT}-${ISO_NOW}" \
```

The `-- CURRENT_STATE.md` before the `-m` flags terminates option parsing; git treats the `-m` flags as pathspecs, not commit message flags. The commit silently fails.

**Evidence of impact:**
- context-repository: 24 passes uncommitted, 10 days since last commit
- command: 9th cycle uncommitted
- atlas: accumulated across 19 idle windows
- supervisor: verified-state.md stale

**Fix:** Move `-- CURRENT_STATE.md` to after all three `-m` flags:

```bash
if git -C "$PROJECT_DIR" commit --only \
     -m "reflect: auto-update CURRENT_STATE.md ${ISO_NOW}" \
     -m "Generated by scripts/lib/reflect.sh after the ${PROJECT} reflection session." \
     -m "session_id=reflect-${PROJECT}-${ISO_NOW}" \
     -- CURRENT_STATE.md \
```
