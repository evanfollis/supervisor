# Patch reflect.sh HEAD-check false positive

## Proposal

**Type:** `reflect.sh` — whitelist autocommit SHAs in HEAD comparison.

**Context:** Each reflection cycle creates one false URGENT file (`URGENT-supervisor-reflection-mutated-head.md`) because the HEAD-check doesn't whitelist autocommit SHAs. This degrades the URGENT signal surface — a reader scanning for real issues hits resolved-issue noise. The fix is to whitelist the SHAs of autocommit commits so they don't trigger false positives.

**Blast radius:** Supervisor. ~15 min implementation.

## Verification before action (required)

- Check `/opt/workspace/supervisor/scripts/lib/reflect.sh` for the HEAD-check logic (guards against reflections that mutate the repo).
- Look for where it compares HEAD before/after and generates the false URGENT file.
- Verify the whitelisting mechanism (likely adding autocommit SHAs to an exclusion list).
- If the fix is already in place, verify:
  - Autocommit SHAs are whitelisted in the HEAD-check
  - No false URGENT files are generated on subsequent reflection cycles
  - Real mutation detections still work (non-autocommit changes still trigger the guard)
- If already implemented, write completion report stating "already landed and verified".

## Acceptance criteria

  - Identify autocommit SHAs (likely from git log with a pattern like "events.jsonl", "autocommit", or a specific commit message pattern)
  - Exclude those SHAs from the mutation check
  - Only report genuine mutations (commits outside the autocommit pattern)
- No false `URGENT-supervisor-reflection-mutated-head.md` files are generated on subsequent cycles
- Real mutations (e.g., a hand-edit to supervisor code) still trigger the guard
