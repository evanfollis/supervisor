# Fix reflect.sh argument ordering

## Summary

The `git commit --only -- CURRENT_STATE.md -m ...` command has argument-order syntax that silently fails. The `--` should come after all `-m` flags, not before. This causes CURRENT_STATE.md to remain uncommitted across all reflected projects. Seven synthesis cycles have correctly diagnosed this. Charter (AGENT.md:508-511) authorizes editing `scripts/lib/` without pre-approval.

## The fix

At `supervisor/scripts/lib/reflect.sh:193`, change:

```bash
if git -C "$PROJECT_DIR" commit --only -- CURRENT_STATE.md \
     -m "reflect: auto-update CURRENT_STATE.md ${ISO_NOW}" \
     -m "Generated by scripts/lib/reflect.sh after the ${PROJECT} reflection session." \
     -m "session_id=reflect-${PROJECT}-${ISO_NOW}" \
```

To:

```bash
if git -C "$PROJECT_DIR" commit --only \
     -m "reflect: auto-update CURRENT_STATE.md ${ISO_NOW}" \
     -m "Generated by scripts/lib/reflect.sh after the ${PROJECT} reflection session." \
     -m "session_id=reflect-${PROJECT}-${ISO_NOW}" \
     -- CURRENT_STATE.md \
```

The sole change: move `-- CURRENT_STATE.md` from before the `-m` flags to after them.
