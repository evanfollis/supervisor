# Fix reflect.sh argument ordering

## Proposal Summary

Correct the git commit argument ordering in `scripts/lib/reflect.sh:193` to resolve uncommitted CURRENT_STATE.md updates across 5 projects over 35+ cycles.

**What:** Move `-- CURRENT_STATE.md` after the `-m` (message) flags in the git commit invocation.

**Status:** Carry-forward from Cycle 53 P1 (unchanged). Charter-authorized, filesystem writable, no principal input needed. CURRENT_STATE.md now uncommitted across 5 projects (atlas: 23 cycles, context-repository: 25+, command: 10+, researcher: 5, valuation: ongoing).

## Context (from synthesis and prior cycles)

The reflection loop auto-updates `CURRENT_STATE.md` in each project repo after a reflection session. The current code attempts to commit this file, but the git command syntax is incorrect: file selection (`-- CURRENT_STATE.md`) appears before the message flags (`-m`). Git interprets this as selecting files to commit BEFORE seeing the message flags, which causes the commit to fail silently or use an incomplete message set.

Correct ordering: `git commit -m "msg1" -m "msg2" -m "msg3" -- CURRENT_STATE.md`

Current (broken): 
```
git -C "$PROJECT_DIR" commit --only -- CURRENT_STATE.md \
     -m "reflect: auto-update CURRENT_STATE.md ${ISO_NOW}" \
     -m "Generated by scripts/lib/reflect.sh after the ${PROJECT} reflection session." \
     -m "session_id=reflect-${PROJECT}-${ISO_NOW}"
```

This has been accumulating across projects and synthesis cycles without resolution, causing: 
- reflection-generated state never auto-committed
- next reflection pass inherits uncommitted state from prior pass
- git HEAD and working tree diverge at CURRENT_STATE.md level
- verification documents show stale committed state while disk has newer changes

