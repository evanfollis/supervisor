# P-self-sustaining-fix — Exclude automation-generated activity from reflect.sh

## Summary

The reflection → synthesis → INBOX pipeline remains functional at every node except consumption. 26 consecutive supervisor reflection cycles with accurate diagnosis and zero conversion to executed changes. 

**Failure class:** Observation system's own artifacts prevent the inactivity short-circuit from firing. `reflect.sh`'s inactivity check counts session-end-hook telemetry events and JSONL files as "activity," so the automated stack continues to generate ~36 Claude Sonnet sessions/day during dormancy (~$2/day, ~$40 cumulative waste).

## Proposed patch

Modify `/opt/workspace/supervisor/scripts/lib/reflect.sh` line 60 to filter out automation-generated telemetry that signals system activity but not user activity:

**Replace line 60:**
```bash
TELEMETRY_COUNT=$(tail -n 5000 "$WORKSPACE_TELEMETRY_DIR/events.jsonl" 2>/dev/null | grep -F "\"$PROJECT\"" | grep -v '"source":"session-end-auto-summary"' | grep -v '"source":"reflect"' | wc -l || true)
```

This excludes telemetry events sourced from:
- `session-end-auto-summary`: automation-generated end-of-session hook events
- `reflect`: reflection-session events themselves

Also modify lines 64-66 to exclude automation-generated JSONL files:

**Replace lines 64-66:**
```bash
JSONL_RECENT=0
if [[ -d "$SESSION_DIR" ]]; then
  JSONL_RECENT=$(find "$SESSION_DIR" -maxdepth 1 -name '*.jsonl' -not -name '*reflect*' -newermt "12 hours ago" 2>/dev/null | wc -l)
fi
```

This filters out reflection session transcripts, which are generated by the automation.

## Impact

- Reduces dormancy sessions from ~36/day to ~4/day across all 8 projects.
- Eliminates false-positive activity detection that triggers expensive reflection cycles during idle periods.

## Blast radius

All 8 projects (automatic). Shared primitive in reflect.sh.
