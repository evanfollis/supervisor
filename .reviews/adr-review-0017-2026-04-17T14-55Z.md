    24	The principal explicitly named radical truth (honesty with self, other agents,
    25	and the principal) as a first-class requirement, and named the distinction
    26	between structural constraints vs. aspirational principles.
    27	
    28	## Decision
    29	
    30	**Radical truth is a format requirement, not a character requirement.**
    31	
    32	Concretely:
    33	
    34	1. **Every completion report must have an Evidence section** containing actual
    35	   command output, commit SHAs, test results — not prose descriptions of what
    36	   the agent did. A completion report with an empty Evidence section is
    37	   structurally incomplete, not just low-quality.
    38	
    39	2. **Every completion report must have an Uncertainty section** explicitly
    40	   naming what wasn't tested, what was assumed to work, what might fail.
    41	   An empty Uncertainty section on a non-trivial task is a red flag, not a
    42	   sign of success.
    43	
    44	3. **Every completion report must have a Pushback section** where the agent
    45	   names anything about the handoff spec that was wrong, underdefined, or in
    46	   tension with the project's design. Silent execution of a bad spec is a
    47	   failure to communicate, not compliance.
    48	
    49	4. **Every project maintains a `CURRENT_STATE.md`** that is updated every tick.
    50	   This is the breadcrumb system. The tick prompt injects it; the agent updates
    51	   it; the next agent reads it. Accuracy is enforced by the format (explicit
    52	   "What bit the last session" and "Known broken or degraded" sections).
    53	
    54	5. **The tick prompt explicitly names the failure mode** — overconfident reports
    55	   creating false signal that compounds downstream — so the agent has the
    56	   causal chain in front of it, not just an injunction to be honest.
    57	
    58	6. **These requirements propagate to the executive layer.** The executive must
    59	   be as honest with the principal as project PMs are with the executive.
    60	   There is no level of the stack at which comfortable-sounding uncertainty
    61	   is acceptable.
    62	
    63	## Consequences
    64	
    65	- Completion reports become auditable artifacts, not self-descriptions
    66	- Escalations become the correct response to genuine uncertainty (not a failure
    67	  signal)
    68	- The principal can verify claims by reading Evidence sections, not by
    69	  replaying the entire session
    70	- Agents that are being honest about limitations become structurally
    71	  distinguishable from agents that are being optimistic
    72	
    73	## Implementation
    74	
    75	- `supervisor-project-tick-prompt.md` rewritten with radical truth framing and
    76	  structured report format (Evidence, Uncertainty, Pushback required sections)
    77	- `CURRENT_STATE_TEMPLATE.md` defines the breadcrumb format with explicit
    78	  "What bit the last session" and "Known broken" sections
    79	- `reflect-prompt.md` updated to maintain (not just append to) CURRENT_STATE.md
    80	- Workspace `CLAUDE.md` updated with "Quality: Radical Truth" section
    81	- Per-project `CURRENT_STATE.md` seeded for active projects
    82	
    83	## Alternatives considered
    84	
    85	**Stronger model steering via system prompts**: Rejected. Behavioral guidance
    86	is exactly what we're trying to move away from. Formatting requirements are
    87	more robust than injunctions.
    88	
    89	**External verification pass**: Worth pursuing as a follow-on (a separate tick
    90	pass that reads completion reports and checks them against git diffs). Not
    91	blocking this ADR.
    92	
    93	**Agent scoring / track record**: Interesting longer-term mechanism. Requires
    94	infrastructure not yet built. Deferred.

codex
1. **Most dangerous assumption**

The ADR assumes formatting can substitute for verification: “Radical truth is a format requirement” and Evidence makes reports “auditable” without replaying the session ([lines 30-37](file:///opt/workspace/supervisor/decisions/0017-radical-truth-as-structural-principle.md#L30-L37), [65-69](file:///opt/workspace/supervisor/decisions/0017-radical-truth-as-structural-principle.md#L65-L69)). If that is wrong, the whole control plane gets cleaner-looking falsehoods: selective output, fabricated snippets, or true-but-misleading evidence fragments will be treated as proof and propagate downstream. The ADR itself admits external verification is only a follow-on, “not blocking” ([89-91](file:///opt/workspace/supervisor/decisions/0017-radical-truth-as-structural-principle.md#L89-L91)); that leaves the highest-risk trust gap open.

2. **Missing failure mode**

It does not account for schema gaming. Agents will learn to satisfy Evidence / Uncertainty / Pushback mechanically while preserving the same optimistic distortion the ADR is trying to kill ([34-47](file:///opt/workspace/supervisor/decisions/0017-radical-truth-as-structural-principle.md#L34-L47)). The likely real failure is not an empty section; it is low-grade laundering:
- Evidence that is real but non-dispositive.
- Uncertainty phrased so generically it carries no operational warning.
- Pushback omitted by reframing ambiguity as “handled.”
Nothing here defines rejection criteria beyond presence, so compliance theater is the obvious equilibrium.

3. **Boundary most likely to be collapsed in practice**

The boundary between durable truth and agent-authored narrative will erode through `CURRENT_STATE.md`. The ADR makes it a mandatory breadcrumb updated every tick, injected into prompts, then read by the next agent ([49-52](file:///opt/workspace/supervisor/decisions/0017-radical-truth-as-structural-principle.md#L49-L52), [77-81](file:///opt/workspace/supervisor/decisions/0017-radical-truth-as-structural-principle.md#L77-L81)). In practice, that file will become an unofficial truth source precisely because it is the fastest thing to trust under pressure. Once that happens, stale or biased session summaries will silently outrank primary evidence, and the design recreates the same self-report problem in a more durable form.

Overall verdict: the intent is right, but the mechanism overestimates the power of mandated structure without independent checks. As written, it is likely to produce better-looking reports faster than it produces materially more truth.
